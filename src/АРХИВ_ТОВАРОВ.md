# Архив товаров

## Функциональность

В системе реализована возможность архивации товаров вместо полного удаления.

## Как это работает

### Для администраторов

1. **Просмотр товаров**
   - Активные товары - доступны для покупки
   - Архивные товары - скрыты от покупки, но сохранены в системе

2. **Переключение между категориями**
   - Кнопка "Активные" - показывает все доступные товары
   - Кнопка "Архивные" - показывает архивированные товары

3. **Управление товаром**
   - Кнопка "Редактировать" - изменить данные товара
   - Меню "⋮" (три точки) - дополнительные действия:
     - **Для активных товаров:**
       - "Переместить в архив" - убрать товар из продажи
       - "Удалить" - удалить товар навсегда (не рекомендуется)
     - **Для архивных товаров:**
       - "Восстановить из архива" - вернуть товар в продажу
       - "Удалить навсегда" - полностью удалить товар

## Визуальные индикаторы

- Архивные товары отображаются с пониженной прозрачностью
- На изображении архивного товара присутствует оранжевый бейдж "Архив"
- Вместо кнопок покупки отображается сообщение "Товар в архиве"

## Рекомендации

✅ **Рекомендуется:**
- Переводить товары в архив при снятии с продажи
- Сохранять историю товаров для аналитики
- Удалять товары только в исключительных случаях

❌ **Не рекомендуется:**
- Удалять товары сразу без архивации
- Удалять товары, по которым были заказы (нарушится история)

## Техническая реализация

### Frontend
- Состояние `showArchived` управляет отображением
- Фильтрация товаров по полю `в_архиве` или `archived`
- DropdownMenu для действий с товаром

### Backend
- Эндпоинт: `PUT /admin/products/:productId/archive`
- Поле: `в_архиве: boolean` и `archived: boolean` (для совместимости)
- Товар остается в базе данных

### Демо режим
- Полная поддержка архивации в localStorage
- Функция `demoArchiveProduct(productId, archive)`
